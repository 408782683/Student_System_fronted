根据要求生成前端，使用VUE3+js+element-plus。

权限说明：分为STUDENT，TEACHER，ADMIN。后端返回的结果如下：
“{
"code": 0,
"message": "string",
"data": {
"id": 0,
"username": "string",
"password": "string",
"role": "ADMIN",
"name": "string",
"phone": "string",
"active": true
}
}”
主页面为学生登录页面，在登录框左下角有“我是管理员”，右下角有“我是老师”，分别跳转到不同的登陆页面。登陆时除了验证获取的验证码，还需通过用户名验证密码与角色，有不同的即提示错误，禁止登录。
要求：
1.登录模块，主要就是分角色进行登录验证，管理员登录验证、教师登录验证、学生登录验证，验证码功能等，从数据库获取相关账号密码信息进行验证，登录成功进入各自不同的页面。 2.管理员模块，包括如下的功能： a) 对新生及刚入职老师的基本信息录入、对已经录入系统的师生信息进行查询、修改、删除。字段要求要有int、字符串、日期、图片等类型。 b) 对学院、专业、班级等信息进行统筹（增删改查），对于外键要有约束，即班级所属专业、专业所属学院。 c) 课程录入、不同学期的专业培养计划进行管理，即在开学前录入不同专业新的一学期需要上哪些课程。课程字段要有课程学分、考核方式、课程分类、课程性质等。 d) 教学任务分配，即给已经分配教学计划的专业分配各课程老师，包括基础课和专业课分类进行信息录入。查询以往教学分配计划。 e) 上传专业及老师课表（专业课表用于给学生下载查看、老师课表用于给老师下载并查看）主要考察文件上传下载功能。 3. 教师模块，包括如下的功能： a) 修改自身的登录密码 b) 可以根据学期查看自己某个学期下的教务分配情况，通俗的来说就是可以查看该学期自己名下有哪些课程要上，以及自己要带哪些班。 c）可以对自己名下课程的学生录入成绩并进行查询。注意分数合法性校验。 d) 可以下载某学期的课表，课表与教师id进行绑定 f) 可以导出行政班的点名册，定名册按照模版进行设计，使用freemarker引擎模版，找到既定格式进行下载。 4. 学生模块，包括如下的功能： a) 修改登录密码，默认密码的修改 b) 可以查看所修学科的成绩，可以打印成绩单 c) 可以下载新学期课表，学生自主选课功能
